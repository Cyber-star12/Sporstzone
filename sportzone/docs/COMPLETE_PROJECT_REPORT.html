ABSTRACT

The Sports Event Registration System, also known as SportZone, is a modern web-based platform developed to simplify and digitize the entire process of managing sports events within colleges and universities. Traditionally, sports event registration has been a manual, time-consuming, and error-prone process. Students had to rely on physical notice boards, paper forms, or scattered online tools to participate in events, which often resulted in confusion and inefficiency.

SportZone addresses these issues by offering a centralized digital platform where students can easily explore various sports events, check details such as venue, date, and availability, and register instantly. The system is designed to be user-friendly, ensuring that even first-time users can navigate it without difficulty. Additionally, the platform enhances communication between organizers and participants by providing real-time updates and notifications.

For administrators, the system acts as a powerful management tool. It allows them to create events, monitor participation, and manage logistics from a single dashboard. By integrating modern web technologies and database systems, SportZone ensures fast performance, scalability, and data reliability. Overall, the system aims to promote active student participation in sports while reducing administrative workload and improving operational efficiency.

The application is built using PHP as the server-side scripting language and MySQL as the database management system. The frontend utilizes Bootstrap 5 framework for responsive design and Bootstrap Icons for visual elements. The system implements robust security measures including CSRF protection, session management, password hashing, and role-based access control to ensure data security and integrity.

Keywords: Sports Event Management, Web Application, PHP, MySQL, Registration System, Admin Dashboard, Bootstrap 5

CHAPTER 1: INTRODUCTION

1.1 Project Introduction

The Sports Event Registration System, popularly known as SportZone, is a comprehensive web-based application designed to automate and streamline the process of managing sports events and participant registrations in educational institutions. This system provides a centralized platform for administrators to create, manage, and monitor sports events while enabling students to easily discover and register for events of their interest.

In traditional educational institutions, sports event management is predominantly carried out using paper-based methods or basic spreadsheet systems. These conventional approaches suffer from numerous limitations including inefficiency in handling large volumes of registrations, difficulty in tracking participant details, lack of real-time updates, and challenges in maintaining accurate records. Furthermore, the manual process of approving or rejecting registrations consumes significant administrative time and is prone to errors and inconsistencies.

SportZone addresses these challenges by providing a fully automated, secure, and user-friendly platform. The application supports two distinct user roles: Students who can browse events, register for events, view their registrations, and cancel registrations if needed; and Administrators who have comprehensive control over event management including creating new events, editing existing events, deleting events, viewing all registrations, approving or rejecting registrations, and exporting registration data to CSV format for further analysis.

The system implements a modern dark-themed sports UI design with gold and orange accents, creating an engaging visual experience for users. It utilizes Bootstrap 5 for responsive design ensuring accessibility across various devices including desktops, tablets, and mobile phones. The backend is powered by PHP 8.0+ with PDO (PHP Data Objects) for secure database operations, while MySQL serves as the reliable database management system.

1.2 Problem Statement

The traditional approach to sports event management in educational institutions faces numerous challenges that the SportZone system aims to address:

1. Paper-Based Registration Process: Traditional registration methods require students to fill out physical forms, which are then manually processed by administrative staff. This process is time-consuming, error-prone, and difficult to manage, especially when dealing with hundreds of participants.

2. Lack of Real-Time Information: Students often remain unaware of upcoming events, registration deadlines, or available slots. There is no centralized platform to broadcast event information effectively.

3. Difficulty in Tracking Registrations: Managing and tracking participant registrations across multiple events becomes increasingly complex with traditional methods. Administrators struggle to maintain accurate records and generate reports.

4. Manual Approval Process: The approval of student registrations requires manual verification and processing, which is both time-consuming and prone to human errors.

5. No Centralized Database: Without a unified system, data redundancy occurs, and maintaining consistency across records becomes challenging.

6. Limited Accessibility: Paper-based systems are not accessible remotely, limiting students' ability to register for events outside of administrative hours.

7. Security Concerns: Manual handling of sensitive student information raises security and privacy concerns. There is no proper mechanism to protect participant data.

1.3 Existing System and Its Limitations

In many educational institutions, sports event registration is still managed through traditional methods such as paper forms, notice boards, or basic online tools like Google Forms. While these methods may seem simple, they come with several significant limitations that affect both students and administrators.

One of the major issues is data redundancy and loss. Paper forms can be misplaced or damaged, and maintaining multiple spreadsheets often leads to duplicated or inconsistent data. This makes it difficult to maintain accurate records.

Another limitation is the lack of real-time tracking. Manual systems do not provide instant updates on the number of available slots, which can result in overbooking or confusion among participants. Students often do not know whether their registration has been confirmed.

The user experience in traditional systems is also poor. There is no centralized dashboard where students can view all available events or track their previous registrations. This lack of visibility reduces engagement and participation.

Additionally, the administrative workload is significantly higher. Organizers must manually verify registrations, assign teams, and compile participant lists, which consumes a lot of time and effort. These inefficiencies highlight the need for a more advanced and automated solution.

1.4 Proposed System

The proposed system, SportZone, is designed to overcome the limitations of existing methods by providing a fully digital and automated solution. It introduces a role-based system where both students and administrators have access to features tailored to their needs.

For students, the platform offers a dynamic dashboard that displays all upcoming sports events along with important details such as date, venue, and available slots. Students can easily browse through events and register with just a few clicks. The system also allows them to track their participation history and receive updates.

For administrators, the system provides a secure control panel where they can manage all aspects of the platform. They can create new events, update existing ones, set participant limits, and monitor registrations in real time. This ensures better organization and efficient event management.

The system uses modern web technologies to provide a responsive and visually appealing interface. It also ensures fast data processing and real-time updates, making the overall experience smooth and efficient. By automating various tasks, the system reduces manual effort and improves accuracy.

1.5 Objectives

The primary objective of the SportZone system is to create an efficient, reliable, and user-friendly platform for managing sports events. The key objectives are as follows:

1. Centralized Platform: To provide a centralized platform where all sports events are listed and managed in one place. This eliminates the need for multiple disconnected systems and ensures that students have easy access to all relevant information.

2. Seamless Registration: To enable students to register for events with minimal effort, ensuring a smooth and hassle-free experience. By incorporating intuitive interfaces and real-time updates, users can quickly check event availability and confirm their participation.

3. Administrative Efficiency: To streamline administrative tasks. Administrators are provided with a dedicated dashboard that allows them to create, update, and monitor events efficiently. This reduces manual work and minimizes the chances of errors.

4. Resource Management: To automatically track the number of participants, team sizes, and available slots, ensuring that events do not exceed their capacity. This prevents overbooking and ensures fair participation.

5. Security and Data Integrity: To implement secure authentication mechanisms and role-based access control, ensuring that sensitive user data is protected and only authorized individuals can access specific functionalities.

5.4 Event Registration Module

Login Page Code (login.php):

<?php
// Handle form submission
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    // Verify CSRF token
    if (!verify_csrf($_POST['csrf_token'] ?? '')) {
        flash('danger', 'Invalid security token.');
    } else {
        $login_type = $_POST['login_type'] ?? 'student';

        if ($login_type === 'admin') {
            // ADMIN LOGIN - Uses email + password
            $email = trim($_POST['email'] ?? '');
            $password = $_POST['password'] ?? '';

            // Use prepared statement - check for admin role
            $stmt = $pdo->prepare('
                SELECT id, name, email, password, role
                FROM users
                WHERE email = ? AND role = "admin"
            ');
            $stmt->execute([$email]);
            $user = $stmt->fetch(PDO::FETCH_ASSOC);

            if ($user && password_verify($password, $user['password'])) {
                session_regenerate_id(true);
                $_SESSION['user_id'] = $user['id'];
                $_SESSION['name'] = $user['name'];
                $_SESSION['email'] = $user['email'];
                $_SESSION['role'] = $user['role'];
                $_SESSION['login_time'] = time();

                header('Location: admin_dashboard.php');
                exit;
            }
        } else {
            // STUDENT LOGIN
            $name = trim($_POST['name'] ?? '');
            $usn = trim($_POST['usn'] ?? '');

            $stmt = $pdo->prepare('
                SELECT id, name, usn, email, role
                FROM users
                WHERE name = ? AND usn = ? AND role = "student"
            ');
            $stmt->execute([$name, $usn]);
            $user = $stmt->fetch(PDO::FETCH_ASSOC);

            if ($user) {
                session_regenerate_id(true);
                $_SESSION['user_id'] = $user['id'];
                $_SESSION['name'] = $user['name'];
                $_SESSION['usn'] = $user['usn'];
                $_SESSION['email'] = $user['email'] ?? '';
                $_SESSION['role'] = $user['role'];
                $_SESSION['login_time'] = time();

                header('Location: dashboard.php');
                exit;
            }
        }
    }
}
?>

APPENDIX A: DATABASE SCHEMA

-- Database: sportzone

CREATE TABLE IF NOT EXISTS users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  email VARCHAR(120) NOT NULL UNIQUE,
  usn VARCHAR(20) NULL,
  password VARCHAR(255) NOT NULL,
  role ENUM('student','admin') NOT NULL DEFAULT 'student',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB;

CREATE TABLE IF NOT EXISTS events (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(150) NOT NULL,
  sport_type VARCHAR(100) NOT NULL,
  event_date DATE NOT NULL,
  event_time TIME NOT NULL,
  venue VARCHAR(150) NOT NULL,
  max_slots INT NOT NULL,
  description TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB;

CREATE TABLE IF NOT EXISTS registrations (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  event_id INT NOT NULL,
  student_name VARCHAR(100) NOT NULL,
  usn_number VARCHAR(20) NOT NULL,
  contact_number VARCHAR(15) NOT NULL,
  course VARCHAR(50) NOT NULL,
  registered_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  approval_status ENUM('pending', 'approved', 'rejected', 'cancelled')
                 NOT NULL DEFAULT 'pending',
  UNIQUE KEY unique_registration (user_id, event_id),
  CONSTRAINT fk_reg_user FOREIGN KEY (user_id) REFERENCES users(id)
                         ON DELETE CASCADE,
  CONSTRAINT fk_reg_event FOREIGN KEY (event_id) REFERENCES events(id)
                          ON DELETE CASCADE
) ENGINE=InnoDB;
